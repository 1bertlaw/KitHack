¿Que es KitHack?

Framework diseñado para automatizar la instalacion de distintas herramientas enfocadas al pentesting. Mediante una recopilacion se obtuvo un listado de las herramientas mas frecuentadas por los amantes de la ciberseguridad, y de la misma forma se obtuvieron herramientas que son poco conocidas pero que funcionan correctamente. Todas estas se clasificaron de la siguiente forma:

  • Android
  • Windows
  • Phishing
  • Wifi Attacks
  • Passwords Attacks
  • Web Attacks
  • Spoofing
  • Information Gathering
  • Others

En cada seccion se encuentran las diferentes herramientas que podemos instalar junto con una pequeña descripcion que nos beneficia en saber de que tratan o que funciones tienen.

¿Como es el proceso de instalacion de las herramientas?

KitHack de manera automatizada se encarga tanto de la clonacion del repositorio como de la instalacion de sus dependencias o requerimientos especiales. Una vez terminada la instalacion de su herramienta, esta se almacenara en la carpeta tools, y dentro de esta carpeta nos vamos a encontrar con otras subcarpetas con los nombres de cada categoria.
Por ejemplo, si la herramienta que usted eligio se encuentra dentro de la categoria Android, esta se almacenara en el directorio KitHack/tools/Android/mi_herramienta.

Si por alguna razon usted inconscientemente trata de volver a instalar una herramienta que ya instalo anteriormente, KitHack va a detectar que esa herramienta ya existe y le arrojara un mensaje indicandole esto mismo y lo regresara al menu principal.

Generador de backdoors.

Una de las funciones mas atractivas que tiene KitHack es el generador de puertas traseras que funciona mediante Metasploit-Framework. Este consiste en la generacion de puertas traseras tanto para Linux, Windows, Android y Mac OS. Tambien contiene 4 formatos multiplataforma los cuales son; PHP, python, bash y perl.

Las 3 principales caracteristicas que tiene este generador de puertas traseras son las siguientes:

  • Unificacion de los 2 tipos de Payloads.
  • Utilizacion de la herramienta Ngrok para generar conexiones TCP (puerto 443 por default).
  • Incorporacion de un nuevo metodo para la certificacion de APKS maliciosas.

Tipos de payloads.

Metasploit en su conjunto de herramientas nos proporciona una gran variedad de payloads los cuales se clasifican por 2 tipos:

  • Payloads por etapas: Este tipo de payload se mantiene de manera muy compacta y solo se encarga de proporcionar los medios para realizar los escenarios de explotacion.
    Cada vez que utilizamos windows/meterpreter/reverse_tcp, linux/meterpreter/reverse_tcp, android/meterpreter/reverse_tcp y entre otros, estamos indicandole a metasploit que prepare un tipo de payload que se divida en 2 etapas. Este en varios escenarios de explotacion puede funcionar muy bien. Sin embargo, existen otros escenarios donde este tipo de payload es lo menos ideal a realizar por el tamaño de memoria que utilizan los bufer y los DLL.
    El ejemplo mas comun de este tipo de escenarios es donde nosotros como atacantes nos encontramos en entornos de bajo ancho de banda o de alta latencia y queremos realizar un ataque fuera de nuestra red local. El costo de datos o tiempo de carga metsrv,stdapi y priv se vuelven dificiles de manejar o practicamente imposibles. En tal escenario, es mejor realizar un ataque de Payload individual.
  • Payloads individuales: Este tipo de payload es un binario que ya incluye todas las partes requeridas de meterpreter, junto con las extensiones requeridas y todas agrupadas en una sola. Cuando se invoca, la instancia de meterpreter ya tiene todo lo que necesita para funcionar y por lo tanto, metasploit no necesita perder tiempo ni ancho de banda para realizar las cargas habituales que se requieren con el enfoque por etapas.

¿Como utilizar meterpreter sin etapas?

En primer lugar, debemos aclarar que los 2 tipos de payloads tienen distinto formato pero esto hace que sean faciles de identificar.

Payload Por etapas Individual
Reverse TCP windows/meterpreter/reverse_tcp windows/meterpreter_reverse_tcp
Reverse HTTPS windows/meterpreter/reverse_https windows/meterpreter_reverse_https
Bind TCP windows/meterpreter/bind_tcp windows/meterpreter_bind_tcp
Reverse TCP IPv6 windows/meterpreter/reverse_ipv6_tcp windows/meterpreter_reverse_ipv6_tcp

Ngrok.

Otra de las caracteristicas de este generador de puertas traseras es la utilizacion de la herramienta Ngrok para generar conexiones TCP. Debemos tener autentificada una cuenta de ngrok en nuestro sistema para que funcione correctamente. Una vez teniendo indicado que tipo de Payload queremos manejar, KitHack nos solicita un LHOST y un LPORT, pero antes de estas 2 solicitudes como primera instancia KitHack nos imprime nuestra IP local, nuestra IP publica y el enlace TCP generado con Ngrok. Este enlace TCP se regenera cada que nos salimos de la herramienta o cada que se ejecuta de nuevo la funcion del menu principal. Si nosotros queremos realizar un ataque fuera de nuestra red local podemos utilizar Ngrok para realizar la conexion TCP con nuestra victima y asi obtener la reverse shell sin ningun tipo de problema.
Ejemplo:

Local IP: 192.168.1.223
Public IP: 201.97.221.230
Ngrok TCP: tcp://0.tcp.ngrok.io:15225

SET LHOST:
SET LPORT:

Para realizar el ataque fuera de nuestra red local, debemos colocar como LHOST: 0.tcp.ngrok.io y como LPORT el puerto que nos genere Ngrok, en mi caso seria el 15225.
Una vez terminado todo el proceso, KitHack nos va a cuestionar si deseamos ejecutar msfconsole, si colocamos que si, automaticamente se va a poner Metasploit en escucha con nuestro localhost y el puerto 443 configurado. Si usted decide no ejecutar msfconsole, la sesion ngrok se perdera y por lo tanto no se va a genenerar la conexion. Recordemos que ngrok debe permanecer activo para que la conexion se pueda realizar y si negamos ejecutar msfconsole KitHack detiene el proceso de ngrok, por lo tanto, si vas a trabajar con ngrok es obligatorio que aceptes ejecutar msfconsole mediante KitHack.

Metodo KitHack para la certificacion de APKS maliciosas.

Hemos llegado a la parte mas interesante de esta documentacion, este nuevo metodo nos permite autentificar la certificacion de una APK maliciosa. ¿Alguna vez han intentado realizar un RAT con el parametro -x de msfvenom? Este metodo mayormente genera errores a los usuarios ya que no admite la mayoria de APKS y tampoco realiza un metodo de certificacion valido que pueda ser indetectable por los antivirus. El metodo de KitHack lo he puesto a prueba en unas 15 aplicaciones como maximo y funciona sin problemas. Hay que aclarar que no todas las aplicaciones van a ser admitidas con este proceso, tales como WhatsApp, facebook, messenger, tiktok, etc.

El proceso que realiza KitHack para estos fines es el siguiente:

  • Genera un payload de forma normal con metasploit.
  • Decompila la APK legitima proporcionada.
  • Decompila el payload que se genero anteriormente.
  • Añade nuevos permisos al archivo AndroidManifest.xml de la APK legitima.
  • Remueve el archivo MainActivity.smali de la carpeta stage del payload generado.
  • Reconfigura los archivos smali del payload generado.
  • Copia la carpeta stage del payload generado a la APK original.
  • Localiza el punto de entrada dentro del archivo AndroidManifest.xml de la APK original.
  • Modifica el archivo .smali del activity para incluir una linea que se encarga de invocar MainService para la sesion meterpreter.
  • Compila nuevamente el APK ya configurado como backdoor.
  • Certifica el archivo APK generado con jarsigner.

En mi repositorio de github se encuentra una tabla de las APKS que llegue a probar con este metodo y que si fueron admitidas de manera correcta, si desea echarles un vistazo da click en "View on GitHub" en la parte superior de la pagina.

Preguntas mas frecuentes.

  1. ¿Como autentifico una cuenta de Ngrok?
    Primero que nada, debemos de registrarnos en la pagina oficial de ngrok con nuestro nombre, correo electronico y una contraseña. Una vez registrados y logueados en la plataforma, debemos ubicar la seccion de Connect your account, ya que en ella se encontrara nuestro token para autentificarnos, copiaremos toda la instruccion del apartado negro a partir de ./ngrok authotoken y lo pegaremos en nuestra terminal en la carpeta KitHack y finalmente ejecutamos.
  2. ¿Por que me sale el error "No module named getch"?
    KitHack esta desarrollado en la version 2.7 de python, por lo tanto, es esencial tener instala esa version para que se pueda ejecutar correctamente. En mi repositorio de Github se encuentran los pasos de instalacion para poder instalar KitHack de manera correcta.
  3. ¿Que tal funciona KitHack en la version 2020 de Kali linux?
    Lamentablemente KitHack esta generando un par de errores en esta nueva version de Kali linux, estos problemas se presentan al momento de ejecutar el instalador y por lo tanto no se puede ejecutar KitHack por la falta de dependencias. Este problema ya lo estoy analizando para asi poder generar una solucion lo antes posible.
